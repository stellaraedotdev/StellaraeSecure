Security Model

StellaraeSecure separates identity, authorization, and panel operations into dedicated services with policy enforcement and auditable decision paths.

Trust boundaries

  • staffdb is the source of truth for accounts and permissions.
  • oauth2 performs authorization decisions and token lifecycle policy.
  • admin is an interface layer and does not own credential material.

Signed staff identity

Sensitive oauth2 endpoints require signed headers for actor identity and timestamp validation, with configurable skew windows.

Step-up protection

High-risk operations require a fresh panel session via x-panel-session-id, reducing replay and privilege escalation risk.

Second-factor policy

Critical admin actions require second-factor readiness from staffdb account records: either TOTP (two_factor_enabled=true) or hardware key bootstrap (hsk_enabled=true) satisfies the policy.
