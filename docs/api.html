API Reference

This reference documents the currently implemented StellaraeSecure APIs with request and response contracts. All endpoints below are derived from the service route handlers in this repository.

Conventions

Content Type

JSON request/response unless noted. Use Content-Type: application/json.

Correlation

oauth2 accepts optional x-correlation-id; if absent, one is generated and propagated to staffdb upstream calls.

Error Response Types

staffdb error payload

{
  "error": "Validation error|Authentication error|Authorization error|Not found|...",
  "status": 400
}

Typical status mapping: 400 validation, 401 authentication, 403 authorization, 404 not found, 429 rate limited.

oauth2 error payload

{
  "error": "validation_error|authentication_error|authorization_error|not_found|upstream_error|internal_error",
  "message": "human-readable message"
}

Service: staffdb

Base URL example: http://127.0.0.1:3000. API routes are under /api.

Authentication and Headers

  • Required header for /api/*: Authorization: Bearer <service_api_key>
  • Mutating endpoints (POST/PATCH/DELETE) also require caller service ID to be listed in PRIVILEGED_SERVICES

Service: oauth2

Base URL example: http://127.0.0.1:4000. oauth2 enforces a combination of admin key, signed staff identity headers, permission claims, and step-up session freshness for high-risk operations.

Admin/Signed Identity Headers

x-admin-key: <admin-api-key>
x-staff-account-id: <account-id>
x-staff-identity-ts: <unix-seconds>
x-staff-identity-sig: <hex-hmac-sha256(account_id:timestamp)>
x-correlation-id: <optional-trace-id>
x-panel-session-id: <required-on-high-risk-operations>

Core Response Types

AccountResponse (staffdb)

{
  "id": "uuid",
  "username": "string",
  "email": "string",
  "is_active": true,
  "account_type": "staff|user",
  "two_factor_enabled": false,
  "created_at": "RFC3339",
  "updated_at": "RFC3339"
}

TokenResponse (oauth2)

{
  "access_token": "string",
  "token_type": "Bearer",
  "expires_in": 900,
  "refresh_token": "string",
  "scope": "openid profile",
  "permissions": ["oauth.client.read"]
}

PanelSessionResponse (oauth2)

{
  "session_id": "uuid",
  "account_id": "uuid",
  "permissions": ["panel.session.verify"],
  "expires_at": "RFC3339"
}
