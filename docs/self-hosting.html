Self-Hosting

Deploy with containers and architecture-aware settings. Use ARM or RISC for test hardware, then move production to AMD64/x86_64.

Default compose profile

docker compose --env-file .env.compose up -d --build

Force ARM test profile

STELLARAE_DOCKER_PLATFORM=linux/arm64 \
  docker compose --env-file .env.compose up -d --build

Production simulation (AMD64)

docker compose --env-file .env.compose \
  -f docker-compose.yml -f docker-compose.prod.yml \
  up -d --build

Multi-arch image publishing

docker buildx build --platform linux/amd64,linux/arm64 \
  -f staffdb/Dockerfile -t ghcr.io/<org>/stellarae-staffdb:latest staffdb --push

docker buildx build --platform linux/amd64,linux/arm64 \
  -f oauth2/Dockerfile -t ghcr.io/<org>/stellarae-oauth2:latest oauth2 --push

docker buildx build --platform linux/amd64,linux/arm64 \
  -f admin/Dockerfile -t ghcr.io/<org>/stellarae-admin:latest admin --push
